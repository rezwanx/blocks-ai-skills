# Workflow Contracts

## Common Headers (all authenticated requests)

```
Authorization: Bearer $ACCESS_TOKEN
x-blocks-key: $X_BLOCKS_KEY
Content-Type: application/json
```

---

## Common Response: BaseResponse

```json
{
  "isSuccess": true,
  "errors": {
    "fieldName": "error message"
  }
}
```

> `errors` is a **dictionary** (key = field name, value = error message), not an array.
> When `isSuccess` is `false`, inspect `errors` to identify which field caused the failure.

## Node Types

Workflows are composed of trigger nodes (exactly one per workflow) and action nodes. Each node has a `type` and an optional `config` object specific to that type.

### WebhookTrigger

Triggers the workflow when an HTTP request hits the generated webhook URL.

```json
{
  "id": "node-1",
  "name": "Webhook Trigger",
  "type": "webhook",
  "position": { "x": 100, "y": 200 },
  "config": {}
}
```

> The webhook URL is generated by the server after the workflow is created and activated. Retrieve it from the workflow details.

---

### EmailTrigger

Triggers the workflow when a new email is received on the configured IMAP mailbox.

```json
{
  "id": "node-2",
  "name": "Email Trigger",
  "type": "emailTrigger",
  "position": { "x": 100, "y": 200 },
  "config": {
    "imapHost": "imap.gmail.com",
    "imapPort": 993,
    "email": "inbox@example.com",
    "password": "app-password",
    "pollInterval": 60
  }
}
```

| Config Field | Type | Required | Notes |
|-------------|------|----------|-------|
| imapHost | string | yes | IMAP server hostname |
| imapPort | integer | yes | IMAP server port (typically 993 for SSL) |
| email | string | yes | Email address to monitor |
| password | string | yes | Email account password or app-specific password |
| pollInterval | integer | no | Polling interval in seconds (default: 60) |

---

### AIAgent

Executes an AI agent with input from previous nodes.

```json
{
  "id": "node-3",
  "name": "AI Agent",
  "type": "aiAgent",
  "position": { "x": 400, "y": 200 },
  "config": {
    "agentId": "agent-id-123"
  }
}
```

| Config Field | Type | Required | Notes |
|-------------|------|----------|-------|
| agentId | string | yes | ID of the AI agent to execute (from `ai-services` domain) |

---

### SendEmail

Sends an email using a pre-configured template.

```json
{
  "id": "node-4",
  "name": "Send Email",
  "type": "sendEmail",
  "position": { "x": 700, "y": 200 },
  "config": {
    "templateId": "template-id-123",
    "to": "{{$json.output.email}}",
    "bodyDataContext": {
      "firstName": "{{$json.output.first_name}}",
      "summary": "{{$node[\"AI Agent\"].json.output.summary}}"
    }
  }
}
```

| Config Field | Type | Required | Notes |
|-------------|------|----------|-------|
| templateId | string | yes | ID of the email template (from `communication` domain) |
| to | string | yes | Recipient email — supports expressions |
| bodyDataContext | object | no | Key/value pairs injected into the template — values support expressions |

---

### HTTPRequest

Makes an HTTP request to an external or internal API.

```json
{
  "id": "node-5",
  "name": "HTTP Request",
  "type": "httpRequest",
  "position": { "x": 400, "y": 400 },
  "config": {
    "method": "POST",
    "url": "https://api.example.com/webhook",
    "headers": {
      "Authorization": "Bearer {{$json.output.token}}",
      "Content-Type": "application/json"
    },
    "body": "{\"data\": \"{{$json.output.result}}\"}",
    "queryParams": {
      "status": "active"
    }
  }
}
```

| Config Field | Type | Required | Notes |
|-------------|------|----------|-------|
| method | string | yes | HTTP method: `GET`, `POST`, `PUT`, `PATCH`, `DELETE` |
| url | string | yes | Target URL — supports expressions |
| headers | object | no | Request headers as key/value pairs — values support expressions |
| body | string | no | Request body (typically JSON string) — supports expressions |
| queryParams | object | no | URL query parameters as key/value pairs — values support expressions |

## Edges

Edges define the connections between nodes. Each edge has a source node and a target node.

```json
{
  "id": "edge-1",
  "source": "node-1",
  "target": "node-3"
}
```

| Field | Type | Required | Notes |
|-------|------|----------|-------|
| id | string | yes | Unique edge identifier |
| source | string | yes | ID of the source node |
| target | string | yes | ID of the target node |

## Expression System

Expressions allow nodes to reference output data from previous nodes in the workflow.

### Reference previous node output

```
{{$json.output.param_name}}
```

References the `param_name` field from the immediately preceding node's output.

### Reference a specific node by name

```
{{$node["Node Name"].json.output.param_name}}
```

References the `param_name` field from a specific named node's output.

### Examples

| Expression | Meaning |
|-----------|---------|
| `{{$json.output.email}}` | Email field from the previous node's output |
| `{{$json.output.body}}` | Body content from the previous node's output |
| `{{$node["AI Agent"].json.output.summary}}` | Summary from the "AI Agent" node's output |
| `{{$node["Webhook Trigger"].json.output.payload}}` | Payload from the webhook trigger's output |

## CreateWorkflowRequest

Used for both creating and updating workflows. Omit `workflowId` to create; include it to update.

```json
{
  "workflowId": "string",
  "name": "My Workflow",
  "description": "Processes incoming webhooks and sends AI-generated emails",
  "nodes": [
    {
      "id": "node-1",
      "name": "Webhook Trigger",
      "type": "webhook",
      "position": { "x": 100, "y": 200 },
      "config": {}
    },
    {
      "id": "node-2",
      "name": "AI Agent",
      "type": "aiAgent",
      "position": { "x": 400, "y": 200 },
      "config": { "agentId": "agent-id-123" }
    }
  ],
  "edges": [
    {
      "id": "edge-1",
      "source": "node-1",
      "target": "node-2"
    }
  ],
  "projectKey": "string"
}
```

| Field | Type | Required | Notes |
|-------|------|----------|-------|
| workflowId | string | no (create) / yes (update) | Omit for create; include to update |
| name | string | yes | Human-readable workflow name |
| description | string | no | Description of the workflow's purpose |
| nodes | array | yes | Array of node objects (at least one trigger node required) |
| edges | array | yes | Array of edge objects connecting nodes |
| projectKey | string | yes | Project identifier from `$PROJECT_SLUG` |

## Execution Status Values

| Status | Meaning |
|--------|---------|
| `pending` | Execution is queued but not yet started |
| `running` | Execution is currently in progress |
| `completed` | All nodes executed successfully |
| `failed` | One or more nodes encountered an error |
| `cancelled` | Execution was manually cancelled |
