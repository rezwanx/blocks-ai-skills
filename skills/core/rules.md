# Global Rules

## General

* Keep skills modular and domain-specific
* Use clear and descriptive naming
* Maintain consistent structure across all features

---

## Actions

* One action = one API call
* Do not combine multiple responsibilities
* Always include curl example
* Always define required authentication

---

## Contracts

* Define request and response format in contracts.md
* Do not assume API structure
* Use contracts as source of truth

---

## Frontend

* Do not use USERNAME, PASSWORD, or tokens in frontend code
* Use backend or proxy if needed
* Handle loading and error states

---

## Error Handling

* Always check API responses
* Handle non-200 responses
* Provide meaningful error messages

---

## Consistency

* Follow naming conventions
* Follow folder structure strictly
* Reuse patterns across all skills
