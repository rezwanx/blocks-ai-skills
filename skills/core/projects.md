# Project Overview

## Purpose

This repository defines a reusable AI skill system for building applications using SELISE Blocks and Claude Code.

It enables structured, AI-assisted development by organizing backend capabilities into modular skills and executable actions.

---

## Stack

* Backend: SELISE Blocks
* AI Runtime: Claude Code
* Frontend: React (Vite) + shadcn/ui

---

## Architecture

* Skills define capabilities (authentication, data, etc.)
* Actions define executable API calls
* Claude Code reads skills and executes actions
* Backend APIs are accessed using ACCESS_TOKEN obtained via identity-access/actions/get-token

---

## Design Principles

* Modular structure by domain
* One action = one responsibility
* Clear separation between frontend and backend
* Secure execution using environment variables

---

## Two Credential Contexts

This system operates in two distinct credential contexts. Claude must always know which context it is in.

### Context 1: Skills Repo (Claude Operations)

Used when Claude executes API calls from within this repository — testing, scaffolding, exploring the backend.

Credentials live in this repo's `.env`:

```
VITE_X_BLOCKS_KEY=...     ← tenant key for the platform being operated
VITE_BLOCKS_OIDC_CLIENT_ID=...
USERNAME=...
PASSWORD=...
ACCESS_TOKEN=      ← populated at runtime by get-token
REFRESH_TOKEN=     ← populated at runtime by get-token
```

### Context 2: Generated Application (App Runtime)

Used by application code generated by Claude. Each app has its own `.env` with its own credentials. The app's `VITE_X_BLOCKS_KEY` may differ if it targets a different tenant.

App `.env` pattern:

```
VITE_VITE_API_BASE_URL=https://api.seliseblocks.com
VITE_VITE_X_BLOCKS_KEY=...     ← that app's tenant key
VITE_VITE_BLOCKS_OIDC_CLIENT_ID=...
```

**Rules:**
* Never use the skills repo `.env` values hardcoded in generated app code
* Generated apps always read from their own `.env` via `import.meta.env.VITE_*`
* When Claude switches to work inside an app directory, it uses that app's `.env`
